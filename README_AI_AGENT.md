# SI-SDMK AI Agent

## Current Status

The active AI path is real n8n integration:

```text
Next.js Chat UI -> /api/ai/chat -> n8n webhook -> Next.js internal tools -> PostgreSQL
```

No mock answer should be used in production. If n8n is not configured, API returns an error instead of fake AI output.

## Required Env

```env
AI_ENABLE_N8N=true
N8N_WEBHOOK_URL=https://n8n.kepegawaian.media/webhook/sisdmk-internal-chat
N8N_PUBLIC_WEBHOOK_URL=https://n8n.kepegawaian.media/webhook/sisdmk-public-chat
N8N_API_SECRET=shared-secret
N8N_WEBHOOK_TIMEOUT_MS=20000
N8N_WEBHOOK_RETRIES=1
```

## Internal Chat Flow

1. User logs in.
2. Frontend sends message and `conversation_id` to `/api/ai/chat`.
3. API validates session and role.
4. API sends user scope to n8n.
5. n8n classifies intent and calls internal tools.
6. Tool API enforces role scope and returns verified data.
7. n8n creates final answer from verified data.
8. API normalizes response and writes `ai_workflow_logs`.

## Public Chat Flow

1. Visitor asks public question on login page.
2. Frontend sends message and `conversation_id` to `/api/ai/public-chat`.
3. n8n may call `/api/internal-ai/tools/public-qna`.
4. Only published QnA may be answered.
5. Private HRIS data must be refused before login.

## Internal Tool Catalog

### Employee Count

```text
POST /api/internal-ai/tools/employee-count
```

Use for totals by status, wilayah, or UKPD. Supports fuzzy filter resolution.

### Search Employee

```text
POST /api/internal-ai/tools/search-employee
```

Use for typo-tolerant employee search by name, NIP, NIK, NRK, status, wilayah, UKPD, or jabatan.

### Employee Profile

```text
POST /api/internal-ai/tools/employee-profile
```

Use for detailed employee sections with masked sensitive identifiers.

### Dashboard Summary

```text
POST /api/internal-ai/tools/dashboard-summary
```

Use for aggregate dashboard data by status, wilayah, and UKPD.

### Public QnA

```text
POST /api/internal-ai/tools/public-qna
```

Use only for public QnA content.

### Pegawai Change Draft

```text
POST /api/internal-ai/tools/pegawai-change-draft
```

Use for AI-assisted CREATE/UPDATE/DELETE draft. This does not directly mutate `pegawai`; it creates an approval task.

Example:

```json
{
  "action": "update",
  "id_pegawai": 123,
  "reason": "User meminta koreksi nama jabatan.",
  "payload": {
    "nama_jabatan_menpan": "Analis SDM Aparatur"
  },
  "user": {
    "id": "1",
    "username": "superadmin",
    "role": "SUPER_ADMIN"
  }
}
```

## n8n Requirements

n8n must:

- Send `x-ai-secret`.
- Return valid JSON.
- Use tool output as ground truth.
- Ask clarification when fuzzy confidence is not high.
- Avoid raw SQL generated by LLM.
- Store conversation memory by `conversation_id`.
- Log request id and tool result.

## Provider Strategy

LLM provider should be configurable:

```env
AI_LLM_PROVIDER=openai
OPENAI_API_KEY=
OPENAI_MODEL=
OLLAMA_BASE_URL=
GEMINI_API_KEY=
```

Provider selection should happen in n8n credentials/workflow settings, not hardcoded in Next.js.
